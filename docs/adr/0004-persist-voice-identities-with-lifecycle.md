# Persist Voice Identities With A Small Lifecycle

Real providers must not generate a different voice for every synthesis request. Orpheus stores a stable local voice identity for each persona and provider. User-provided `voice.assets` take precedence. If no assets are present, a persona-capable provider may generate or materialize a voice identity once and reuse it.

Voice identities are runtime assets and must stay out of Git. They are stored under an ignored runtime folder such as `.orpheus/voices`.

The lifecycle is intentionally small:

- `active`: used for normal synthesis
- `candidate`: generated by voice regeneration and awaiting acceptance
- `rejected`: no longer wanted and eligible for cleanup

The first `speak` call may create and activate a voice identity automatically. `voice regenerate <personaId>` creates a candidate and a preview audio file. `voice accept <personaId> <candidateId>` makes the candidate active and may clean up old candidates. `voice reject <personaId> <candidateId>` marks a candidate as rejected.

Voice identities include a fingerprint from persona id, provider, voice id, voice style, relevant provider settings, and local asset metadata. For V1, local asset metadata means normalized path, file size, and modified time rather than full hashes of large files. If the current persona no longer matches the active voice fingerprint, the voice is stale. V1 may continue to use it, but must warn clearly.

Regeneration preview text is selected in this order:

1. explicit text supplied by the user
2. last original synthesis text for the persona
3. persona `previewText`
4. global preview text from app configuration
5. clear error

Only the last original synthesis text is stored. The transformed persona output is not stored for regeneration preview.

**Consequences**

Voice generation work is done once per accepted voice identity rather than on every request. Users can reroll voices without losing accepted voices immediately. Audio cache keys must include the active voice identity version or fingerprint so cached audio from an old voice is not reused after a voice change.
